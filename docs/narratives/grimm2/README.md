# Grimm 2.0 — Workflow (docs-first)

Dieses Verzeichnis enthält ein Template, das **Moral als Slice** (vom Leser erzeugt) und **Referenzstab durch Stacking** (Invarianten über mehrere Stories) unterstützt.

## Datei-Schichten (Stratigrafie)
1) **RAW**: unveränderte Quelle (Chat/Export/Notiz)
2) **EXTRACT**: markierte Passagen + Claim-Tags + Trigger ("wenn X, dann Y")
3) **SYNTH**: begehbare Treppe (Narration 2.0), die auf RAW/EXTRACT verweist

## Schreiben einer Grimm-2.0 Story
- Nutze `_template.md` als Startpunkt (kopieren/umbenennen).
- Fülle **Motive** als Funktionen (nicht als Deko).
- Baue 5–7 **Stufen**; jede Stufe endet mit einem Refrain-Fragment.
- Formuliere 3–7 **Trigger** als beobachtbare Kipp-Punkte.
- Lasse die **Slice-Zone** bewusst offen (sie gehört dem Leser).

## Stacking (Referenzstab)
Wenn mehrere Stories dieselben Motive/Trigger wiederholen, werden **Invarianten-Kandidaten** sichtbar.
Diese Invarianten sind der "Referenzstab" für neue Situationen.

## Navigation
- Jede Story soll im Abschnitt „Pointers" auf RAW/EXTRACT zeigen (relative Pfade).
- Der Pointer `index/pointers/grimm2__template.pointer.md` ist der Einstiegspunkt für das Template.
