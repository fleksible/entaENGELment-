# Grimm 2.0 — Narrative Template (docs-first)

> Refrain: **Schichtung speichert. Erosion liest. Kante wird Oberfläche.**

## 0 — Ausgangslage (nur 3–5 Sätze)
[FACT] Kontext/Ort/Zeit (minimal)
[MET] Das Bild, das die Szene trägt

## 1 — Motive (5–9 Koordinaten)
Liste Motive als Funktionen, nicht als Deko:
- Grenze/Front:
- Wirbel/Helizität:
- Opfer/Sediment:
- Treppe/Schichtung:
- Gate:
- Ticket/Consent:
- Receipt/Trace:
- KillSwitch (falls relevant):
- (optional) Trickster/Greeter:

## 2 — Treppe (Stufen 1–7)
Jede Stufe 2–6 Zeilen. Jede Stufe endet mit einem Refrain-Fragment.
Stufe 1: …
> Refrain: …
Stufe 2: …
> Refrain: …
…
Stufe 7: …
> Refrain: …

## 3 — Trigger (Wenn–Dann, 3–7 Stück)
Formuliere beobachtbare Kipp-Punkte:
- Wenn ____ passiert → dann ____.
- Wenn ____ überlappt/berührt (RCC8) → dann ____.
- Wenn Leakage droht → dann ____ (Guard).

## 4 — Slice-Zone (vom Leser auszufüllen)
Diese Zone NICHT beantworten. Sie ist der "Schnitt".
- Welche Grenze war entscheidend?
- Was wurde verdichtet, um Fundament zu werden?
- Wo war das Gate – und wodurch wurde es passiert?
- Was wäre der KillSwitch in dieser Story?

## 5 — Stack-Anchor (1–3 Invarianten-Kandidaten)
Nur Hinweise, keine Moral:
- Invariante-Kandidat A: …
- Invariante-Kandidat B: …
- Invariante-Kandidat C: …

- RAW (falls vorhanden): …
- EXTRACT (falls vorhanden): …
- SYNTH (diese Datei): …
