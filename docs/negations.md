## Einleitung

Dieses Dokument definiert EntaENGELment durch Negation. Es beschreibt, was das Projekt **nicht** ist, **nicht** behauptet und **nicht** leisten kann. Diese negative Theologie ist keine Schwäche, sondern präzise Grenzziehung.

## EntaENGELment ist NICHT

### Kein Produkt

```
❌ EntaENGELment ist KEIN fertiges Produkt
❌ EntaENGELment ist KEINE marktreife Software
❌ EntaENGELment ist KEIN Tool für Endanwender

✓ EntaENGELment IST ein Research-Prototyp
✓ EntaENGELment IST ein Explorations-Raum
✓ EntaENGELment IST ein Living Document
```

**Implikation:** Keine Garantien für Stabilität, API-Konsistenz oder Backwards-Compatibility.

### Keine Physik-Theorie

```
❌ EntaENGELment behauptet KEINE physikalischen Mechanismen
❌ EntaENGELment ist KEINE Quantenmechanik-Erweiterung
❌ EntaENGELment macht KEINE empirisch testbaren Vorhersagen

✓ EntaENGELment NUTZT Metaphern aus der Physik
✓ EntaENGELment EXPLORIERT strukturelle Analogien
✓ EntaENGELment IST ein konzeptuelles Framework
```

**Implikation:** Begriffe wie "Verschränkung", "Resonanz", "Feld" sind **Metaphern**, keine physikalischen Behauptungen.

### Kein EPR-Verschränkungs-Beweis

```
❌ EntaENGELment beweist NICHT, dass Kommunikation "verschränkt" ist
❌ EntaENGELment ist KEIN Experiment zur Nicht-Lokalität
❌ EntaENGELment widerlegt KEINE Bell-Ungleichungen

✓ EntaENGELment NUTZT die Struktur von Verschränkung als Denkmodell
✓ EntaENGELment EXPLORIERT Analogien zwischen Quantenkorrelationen und Bedeutung
✓ EntaENGELment IST ein Modell, keine Messung
```

**Implikation:** Wenn wir "Verschränkung" sagen, meinen wir **strukturelle Ähnlichkeit**, nicht **physikalische Identität**.

### Kein Peer-Review-Ersatz

```
❌ EntaENGELment ist NICHT peer-reviewed
❌ EntaENGELment ersetzt KEINE wissenschaftliche Validierung
❌ EntaENGELment behauptet KEINE wissenschaftliche Autorität

✓ EntaENGELment IST Pre-Paper-Exploration
✓ EntaENGELment IST ein Workspace für Ideen
✓ EntaENGELment KÖNNTE Grundlage für Papers werden
```

**Implikation:** Zitiere EntaENGELment nicht als wissenschaftliche Quelle ohne Peer-Review.

### Nicht für Laien

```
❌ EntaENGELment ist NICHT selbsterklärend
❌ EntaENGELment hat KEINE flache Lernkurve
❌ EntaENGELment ist KEIN Tutorial

✓ EntaENGELment SETZT Vorwissen voraus
✓ EntaENGELment HAT eine steile Lernkurve
✓ EntaENGELment IST für Eingeweihte
```

**Implikation:** Frustration bei Neulingen ist erwartbar. Das ist kein Bug, sondern Designentscheidung.

## Kenogrammatik ist NICHT

### Keine vollständige Logik

```
❌ Kenogrammatik ist KEIN Ersatz für formale Logik
❌ Kenogrammatik ist NICHT widerspruchsfrei beweisbar
❌ Kenogrammatik ist KEINE abgeschlossene Theorie

✓ Kenogrammatik IST eine Notation für Lücken
✓ Kenogrammatik MARKIERT Stellen der Unbestimmtheit
✓ Kenogrammatik IST ein Tool, keine Wahrheit
```

**Implikation:** Kenogramme sind **Zeiger auf Leerstellen**, nicht Füllungen.

### Kein Wissensersatz

```
❌ Kenogrammatik ERSETZT KEIN Wissen
❌ Kenogrammatik GENERIERT KEINE Erkenntnisse
❌ Kenogrammatik FÜLLT KEINE Lücken

✓ Kenogrammatik MARKIERT Nicht-Wissen
✓ Kenogrammatik ZEIGT wo Fragen offen sind
✓ Kenogrammatik IST eine Landkarte der Ignoranz
```

**Implikation:** Ein Kenogramm ist eine **Frage**, keine **Antwort**.

### Keine mystische Lehre

```
❌ Kenogrammatik ist KEINE Esoterik
❌ Kenogrammatik beruft sich NICHT auf verborgenes Wissen
❌ Kenogrammatik ist NICHT "nur für Eingeweihte verstehbar"

✓ Kenogrammatik IST operational definiert
✓ Kenogrammatik IST lernbar
✓ Kenogrammatik IST kritisierbar
```

**Implikation:** Wenn etwas unklar ist, liegt es an der Dokumentation, nicht an der "Tiefe".

## Receipts sind NICHT

### Keine Blockchain

```
❌ Receipts sind KEINE distributed ledger
❌ Receipts sind NICHT dezentral verifizierbar
❌ Receipts nutzen KEINEN Proof-of-Work/Stake

✓ Receipts NUTZEN HMAC-Signaturen
✓ Receipts SIND lokal verifizierbar
✓ Receipts SIND ein Audit-Trail, kein Konsens-Mechanismus
```

**Implikation:** Receipts sind so sicher wie das Secret. Kein Secret = kein Vertrauen.

### Nicht tamper-proof

```
❌ Receipts sind NICHT kryptographisch tamper-proof
❌ Receipts VERHINDERN KEINE Manipulation
❌ Receipts GARANTIEREN KEINE Integrität

✓ Receipts DETEKTIEREN Manipulation (wenn Secret intakt)
✓ Receipts DOKUMENTIEREN Zustand zu einem Zeitpunkt
✓ Receipts SIND so stark wie das schwächste Glied
```

**Implikation:** Secret-Leak kompromittiert **alle** Receipts. Rotation ist kritisch.

### Nicht rechtlich bindend

```
❌ Receipts sind KEINE Verträge
❌ Receipts ERSETZEN KEINE rechtliche Dokumentation
❌ Receipts SIND NICHT vor Gericht verwendbar

✓ Receipts SIND ein interner Audit-Trail
✓ Receipts DIENEN der Nachvollziehbarkeit
✓ Receipts SIND ein Gedächtnis, kein Vertrag
```

**Implikation:** Für rechtliche Zwecke: echte Dokumentation verwenden.
